_Pourquoi est-il préférable de lancer les tests dans Docker plutôt que sur votre machine directement lors d'un pipeline CI ?_

Il est préférable de lancer les tests dans Docker lors d'un pipeline CI car ça garantit un environnement identique et isolé à chaque exécution.

**Quel est l'avantage de la méthode "override" (docker-compose.yaml + docker-compose.prod.yaml) plutôt que 2 fichiers indépendants ?**

L'avantage est qu'on évite de dupliquer la configuration de base. Le fichier prod.yaml ne contient que les différences (ajouts, modifications), ce qui rend le code plus maintenable et évite les erreurs de synchronisation.

**Pourquoi ajouter Prometheus dans prod.yaml plutôt que dans docker-compose.yaml de base ?**

Car Prometheus est utile uniquement en production ou pour la supervision. En développement local, ça alourdit l'environnement et ce n'est pas nécessaire.

**Première alerte Grafana à mettre sur un VPS de production ?**

Une alerte sur l’utilisation CPU/RAM ou l’espace disque (surtout disque).
Exemple : alerte si le disque dépasse 80–90 %.

**Qu'est-ce qu'une SBOM et à quoi ça sert ?**

SBOM = Software Bill of Materials. C'est une liste de tous les composants logiciels (dépendances, libraries) utilisés dans un projet. Ça sert à connaître exactement quelles versions sont présentes, pour la sécurité et la conformité.
